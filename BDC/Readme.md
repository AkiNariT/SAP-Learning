## BDC：SAP Business Data Cloud<br>

## BDC是想掌握企业数据
因为总览了企业内部各系统数据，统一了说法，处理等。<br>
<br>
以往常见方式，企业内部不同软件协作。通过接口链接互相传递数据。<br>
在做数据分析时需要挨个抽出，总结来做模型等一些列东西才可以开始分析。<br>
但BDC不同，有了BDC，可以将所有数据送往BDC，这会使得所有数据格式统一，也会让处理变得统一。<br>
把复杂的多程序用语变成BDC统一用语然后统一处理，便于分析。<br>
但也只是数据分析，不影响程序间的实时处理。<br>

## 简单理解
BDC 是企业数据、分析、的数据平台。<br>

因为是在数据分析，所以后期分析出结果是可以用作AI训练的。<br>
因为AI训练是需要统一提示词，BDC属于AI训练初期的数据整合阶段，是方便AI模型训练。<br>
所以 BDC 官方定位更偏向 统一和治理 SAP 与非 SAP 数据，支撑高级分析和AI数据分析。<br>
<br>

做 BDC 项目时，可能做：
- SAP Datasphere 建模
- SAP BW 数据迁移/现代化
- SAC 报表分析
- 数据产品设计
- SAP Databricks 数据处理
- 给 AI/分析提供统一数据基础


以上结论来看BDC就是数据分析，太过概念。<br>
具体的操作学习才能看到是否有作用。<br>
<br>
从目前来看，最终落地的将会是一个可实行的数据模型，或者说数据结构。<br>
或者是业务部门的报表画面。<br>
权限设计,类似公司下属哪个工厂只能看和其工厂产品相关的销售数据。<br>

所以工作流程应该是。<br>
报表需求 -> 找寻各个系统的数据来源 -> 数据表之间抽取逻辑 -> 数据建模（重点） -> 测试等
